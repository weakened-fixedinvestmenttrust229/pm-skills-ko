# /research-users -- 사용자 리서치 종합

원시 리서치 데이터를 실행 가능한 사용자 페르소나, 행동 세그먼트, 고객 여정 지도로 전환합니다. 설문 데이터, 인터뷰 노트, 피드백, 분석 데이터, 또는 탐색적 리서치를 위한 제품 설명을 수용합니다.

## 실행 예시

```
/research-users [설문 결과, 인터뷰 노트, 또는 피드백 데이터 업로드]
/research-users B2B project management tool for agencies — help me understand our users
/research-users [사용자 피드백 또는 지원 티켓 데이터 붙여넣기]
```

## 워크플로

### Step 1: 리서치 입력 수집

다음의 조합을 허용합니다:
- 설문 응답 (CSV, 스프레드시트, 붙여넣기)
- 인터뷰 노트 또는 녹취록
- 지원 티켓 또는 기능 요청
- 제품 분석 / 행동 데이터
- NPS 또는 만족도 데이터
- 제품 설명 (데이터 없이 탐색적 리서치 수행 시)

다음을 확인합니다:
- 어떤 리서치 자료가 있나요? 형식은 무엇인가요?
- 무엇을 이해하고 싶으신가요? (우리 사용자가 누구인지, 어떻게 다른지, 어디서 마찰이 발생하는지)
- 이 정보가 어떤 결정에 활용될 예정인가요? (로드맵, 포지셔닝, 가격, 온보딩)

### Step 2: 페르소나 구축

**user-personas** 스킬을 적용합니다:

- 데이터에서 3~4개의 뚜렷한 페르소나 식별
- 각 페르소나: 이름, 역할, 목표(JTBD), 페인 포인트, 이점, 행동 패턴
- 예상치 못한 인사이트 포함 — 데이터에서 놀라운 점
- 페르소나 비율 표시 (데이터가 허용하는 경우, 전체 사용자 기반에서 차지하는 %)

### Step 3: 사용자 세분화

**user-segmentation** 및 **market-segments** 스킬을 적용합니다:

- 행동 기반 세그먼트 생성 (인구통계만이 아닌)
- 각 세그먼트: 규모, JTBD, 제품 적합도, 지불 의향, 참여 수준
- 가장 높은 가치를 지닌 세그먼트와 가장 높은 성장 세그먼트 식별
- 세그먼트와 페르소나 매핑 (겹치는 부분)

### Step 4: 고객 여정 지도 작성

**customer-journey-map** 스킬을 적용합니다:

- 엔드투엔드 여정 지도 작성: 인지 → 고려 → 온보딩 → 적극적 사용 → 확장 → 지지
- 각 단계: 접점, 감정, 페인 포인트, 아하 모먼트
- 가장 큰 이탈 지점 식별
- 더욱 강화할 가치 있는 기쁨의 순간 부각

### Step 5: 리서치 보고서 작성

```
## 사용자 리서치 보고서: [제품]

**날짜**: [오늘]
**데이터 소스**: [분석한 내용]
**샘플 크기**: [해당하는 경우]

### 경영진 요약
[핵심 발견 및 시사점: 3~5문장]

### 페르소나

#### 페르소나 1: [이름] — "[그들을 표현하는 인용문]"
- **누구**: [역할, 맥락, 경험 수준]
- **주요 JTBD**: [When..., I want to..., so I can...]
- **핵심 페인 포인트**: [상위 3개]
- **핵심 이점**: [기쁨을 주는 것]
- **행동 패턴**: [제품 사용 방식]
- **비율**: [전체 사용자 기반의 X%]

[각 페르소나에 대해 반복]

### 사용자 세그먼트
| 세그먼트 | 규모 | 주요 JTBD | 제품 적합도 | 가치 | 성장 |
|---------|------|---------|----------|-----|-----|

### 고객 여정 지도
| 단계 | 접점 | 감정 | 페인 포인트 | 기회 |
|-----|-----|-----|---------|-----|

### 핵심 인사이트
1. [근거 데이터를 포함한 인사이트]
2. ...

### 권고 사항
1. [발견에 연결된 실행 가능한 권고]
2. ...

### 미해결 질문
[데이터가 답하지 못한 것 — 후속 리서치 제안]
```

마크다운으로 저장합니다.

### Step 6: 다음 단계 제안

- "특정 페르소나를 더 깊이 파악하기 위한 **인터뷰 스크립트를 작성**해 드릴까요?"
- "이 세그먼트들의 **감성을 분석**해 드릴까요?"
- "상위 페르소나를 위한 **가치 제안을 구축**해 드릴까요?"
- "여정 지도의 페인 포인트를 **기능 기회로 우선순위화**해 드릴까요?"

## 참고 사항

- 데이터가 부족한 경우 신뢰 수준에 대해 명확히 표시 — 인터뷰 5건은 결론이 아닌 가설
- 페르소나는 장식이 아닌 실용적이어야 함 — 모든 페르소나는 제품 결정에 영향을 미쳐야 함
- 행동 기반 세그먼트가 제품 결정에 있어 인구통계 기반 세그먼트보다 더 실행 가능
- 여정 지도는 행동뿐 아니라 감정을 표면화해야 함 — 사용자가 좌절감을 느끼는 곳과 기쁨을 느끼는 곳이 우선순위 결정을 이끔
- 데이터가 없는 경우, 리서치 기반 가설을 생성하고 검증 방법을 권고
