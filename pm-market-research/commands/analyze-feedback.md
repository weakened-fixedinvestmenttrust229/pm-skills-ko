# /analyze-feedback -- 사용자 피드백 분석

대량의 사용자 피드백(리뷰, 설문, 지원 티켓, NPS 응답)을 감성 분석 및 세그먼트별 패턴을 포함한 구조화된 인사이트로 처리합니다.

## 실행 예시

```
/analyze-feedback [NPS 응답 CSV 업로드]
/analyze-feedback [앱스토어 리뷰 또는 설문 응답 붙여넣기]
/analyze-feedback [지원 티켓 내보내기 업로드]
```

## 워크플로

### Step 1: 피드백 데이터 수집

다음 형식을 모두 허용합니다:
- 피드백 텍스트가 포함된 CSV/Excel (선택적 메타데이터: 날짜, 세그먼트, 평점)
- 붙여넣은 텍스트 (리뷰, 설문 응답, Slack 메시지)
- 피드백 도구에서 업로드한 문서 또는 내보내기 파일

다음을 확인합니다:
- 이 피드백의 종류는 무엇인가요? (NPS, 리뷰, 지원 티켓, 설문 등)
- 별도로 분석할 세그먼트가 있나요? (사용자 등급, 플랜, 지역)
- 무엇을 찾고 있나요? (일반 테마, 특정 이슈, 시간에 따른 트렌드)

### Step 2: 분석

**sentiment-analysis** 스킬을 적용합니다:

- **감성 점수 매기기**: 각 피드백을 분류 (긍정, 중립, 부정)
- **테마 추출**: 반복되는 주제를 식별하고 관련 피드백 클러스터링
- **빈도 분석**: 각 테마의 등장 횟수 집계
- **세그먼트 분석**: 사용자 세그먼트별 감성과 테마 분류 (데이터가 있는 경우)
- **트렌드 감지**: 날짜 데이터가 있는 경우, 시간에 따른 감성 변화 식별

### Step 3: 분석 보고서 작성

```
## 피드백 분석 보고서

**날짜**: [오늘]
**분석된 피드백**: [건수] 개
**출처**: [NPS 설문 / 앱 리뷰 / 지원 티켓 등]
**기간**: [날짜 범위 (있는 경우)]

### 전체 감성
- 긍정: [X%] | 중립: [Y%] | 부정: [Z%]
- 평균 감성 점수: [X/10]
- 트렌드: [개선 / 안정 / 악화]

### 주요 테마
| # | 테마 | 언급 수 | 감성 | 가장 영향받은 세그먼트 |
|---|------|---------|------|----------------------|

### 테마 심층 분석

#### 테마 1: [이름] — [X]건, [감성]
- **사용자가 말하는 것**: [대표 인용문을 포함한 요약]
- **근본 원인**: [이 피드백을 유발하는 요인]
- **영향**: [리텐션, 만족도, 또는 매출에 미치는 영향]
- **권고 사항**: [대응 방안]

[상위 5~8개 테마에 대해 반복]

### 세그먼트 분석
| 세그먼트 | 건수 | 평균 감성 | 주요 테마 | 주요 차이점 |
|---------|------|---------|---------|-----------|

### 주목할 인용문
> "[인용문]" — [세그먼트, 감성]

### 시간별 트렌드
[날짜 데이터가 있는 경우: 감성 변화를 보여주는 차트용 데이터]

### 실행 가능한 인사이트
1. [인사이트 + 권고 조치]
2. ...

### 데이터 공백
[이 피드백에서 알 수 없는 것 — 후속 리서치 제안]
```

마크다운으로 저장합니다. 입력이 구조화된 데이터(CSV)인 경우, 감성 점수가 추가된 데이터도 CSV로 저장합니다.

### Step 4: 다음 단계 제안

- "이 피드백 패턴에서 **사용자 페르소나를 생성**해 드릴까요?"
- "**상위 테마를 피처 요청으로 분류**해 드릴까요?"
- "특정 테마를 더 깊이 파악하기 위한 **인터뷰 스크립트를 작성**해 드릴까요?"

## 참고 사항

- 감성 분석은 근사값이므로 엣지 케이스(비꼼, 혼합 감성, 비영어 텍스트)는 별도 표시
- 테마 추출은 표면적인 주제가 아닌 요청 뒤에 있는 니즈를 파악해야 함
- 세그먼트별 샘플 크기가 작으면 낮은 신뢰도를 명시
- NPS 데이터의 경우 비추천자(0~6), 수동적 응답자(7~8), 추천자(9~10)를 별도 분석
- 입력이 구조화된 데이터인 경우, 사용자가 자체 도구에서 활용할 수 있도록 보강된 CSV 출력
