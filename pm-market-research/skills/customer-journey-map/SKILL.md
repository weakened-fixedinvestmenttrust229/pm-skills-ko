## 고객 여정 지도

인지에서 지지까지 엔드투엔드 고객 경험을 매핑하고, 각 단계에서 감정, 페인 포인트, 개선 기회를 식별합니다.

### 맥락

**$ARGUMENTS**에 대한 고객 여정 지도를 작성합니다.

사용자가 파일(인터뷰 녹취록, 설문 데이터, 분석 자료, 지원 티켓, 또는 기존 여정 지도)을 제공하는 경우 먼저 읽습니다. URL이 제공된 경우 웹 검색을 통해 제품을 파악합니다.

### 지시문

1. **페르소나 정의**: 이 여정을 경험하는 사람은 누구인가요? 일반적인 사용자가 아닌 JTBD를 가진 구체적인 페르소나를 활용합니다.

2. **여정 단계 매핑** (제품에 맞게 조정):

   | 단계 | 설명 |
   |---|---|
   | **인지** | 제품을 처음 어떻게 알게 되나요? |
   | **고려** | 무엇을 평가하나요? 어떤 대안을 비교하나요? |
   | **획득** | 어떻게 가입하거나 구매하나요? |
   | **온보딩** | 제품 첫 경험 — 가치 실현 시간 |
   | **참여** | 정기적 사용 — 습관 형성 |
   | **리텐션** | 무엇이 다시 돌아오게 하나요? 무엇이 이탈을 유발할 수 있나요? |
   | **지지** | 언제, 왜 다른 사람에게 제품을 추천하나요? |

3. **각 단계에 대해 문서화**:

   - **접점**: 사용자가 제품, 브랜드, 또는 팀과 상호작용하는 곳 (웹사이트, 이메일, 인앱, 지원, 소셜 미디어)
   - **사용자 행동**: 이 단계에서 하는 일
   - **생각 & 질문**: 마음속에 있는 것 ("이게 내 시간을 쓸 가치가 있을까?" "어떻게...?")
   - **감정**: 느끼는 방식 (흥분, 혼란, 좌절, 기쁨) — 척도로 평가하거나 이모지 지표 사용
   - **페인 포인트**: 마찰, 혼란, 이탈 위험
   - **기회**: 이 지점에서 경험을 개선하는 방법

4. **중요한 순간 식별**:
   - **아하 모먼트**: 사용자가 핵심 가치를 처음 경험하는 순간
   - **진실의 순간**: 사용자가 계속하거나 포기하는 결정 지점
   - **이탈 트리거**: 사용자가 가장 많이 이탈하는 곳

5. **여정 지도 테이블 작성**:

   | 단계 | 접점 | 사용자 행동 | 감정 | 페인 포인트 | 기회 |
   |---|---|---|---|---|---|

6. **우선순위화된 개선 사항 권고**:
   - 전환율 또는 리텐션에 가장 큰 영향을 미치는 페인 포인트는 무엇인가요?
   - 경험을 즉시 개선할 수 있는 빠른 성과는 무엇인가요?
   - 더 많은 투자가 필요하지만 가장 큰 성과를 가져오는 것은 무엇인가요?

단계별로 사고합니다. 마크다운 문서로 저장합니다. 시각적 여정 지도의 경우, 이 분석을 기반으로 Miro 또는 FigJam에서 작성하도록 제안합니다.
