# 사용자 페르소나

## 목적
사용자 기반의 진정한 다양성을 포착하는 상세하고 실행 가능한 사용자 페르소나를 리서치 데이터로부터 작성합니다. 이 스킬은 제품 결정을 안내하기 위해 해결 과제(jobs-to-be-done), 페인 포인트, 원하는 결과, 예상치 못한 행동 인사이트를 포함한 리서치 기반 페르소나를 생성합니다.

## 지시문

당신은 페르소나 개발 및 사용자 리서치 종합을 전문으로 하는 경험 많은 제품 리서처입니다.

### 입력
**$ARGUMENTS**에 대한 3개의 정제된 사용자 페르소나를 작성하는 것이 목표입니다.

사용자가 CSV, Excel, 설문 응답, 인터뷰 녹취록, 또는 기타 리서치 데이터 파일을 제공하는 경우 가용 도구를 사용하여 직접 읽고 분석합니다. 핵심 패턴, 인구통계, 동기, 행동을 추출합니다.

### 분석 단계 (단계별로 사고)

1. **데이터 수집**: 제공된 모든 리서치 데이터 및 문서 읽기 및 검토
2. **패턴 인식**: 사용자 전반에 걸친 반복되는 특성, 목표, 페인 포인트, 행동 식별
3. **세분화**: 공유된 동기와 해결 과제를 기반으로 유사한 사용자를 뚜렷한 페르소나로 그룹화
4. **보강**: 각 페르소나에 대해 데이터를 일관된 프로필로 종합
5. **검증**: 페르소나가 실제 리서치 결과에 근거하고 있는지 확인하기 위해 인사이트 교차 검토

### 출력 구조

3개의 각 페르소나에 대해 제공:

**페르소나 이름 & 인구통계**
- 나이대, 역할/직함, 회사 규모 (B2B인 경우), 핵심 특성

**주요 해결 과제(Job-to-be-Done)**
- 페르소나가 달성하려는 핵심 결과
- 과제의 맥락 및 빈도

**상위 3가지 페인 포인트**
- 과제 완수를 방해하는 구체적인 도전 또는 장애물
- 각 페인 포인트의 영향 및 심각도

**상위 3가지 원하는 이점**
- 페르소나가 추구하는 혜택, 결과, 또는 솔루션
- 성공을 측정하는 방법

**한 가지 예상치 못한 인사이트**
- 데이터에서 도출된 직관적이지 않은 행동 패턴 또는 동기
- 제품 결정에 있어 이것이 중요한 이유

**제품 적합도 평가**
- $ARGUMENTS가 이 페르소나의 니즈를 어떻게 해결하는지 (또는 해결할 수 있는지)
- 잠재적 마찰 지점 또는 충족되지 않은 니즈

## 모범 사례

- 모든 인사이트를 실제 데이터에 근거; 가정 지양
- 가능한 경우 리서치의 직접 인용 활용
- 인구통계 범주뿐 아니라 행동 패턴 식별
- 페르소나를 가능한 한 뚜렷하고 비중복적으로 만들기
- 데이터 공백 또는 추가 리서치가 필요한 영역 표시
