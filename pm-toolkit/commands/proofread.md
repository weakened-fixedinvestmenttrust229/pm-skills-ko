# /proofread -- 문법 및 흐름 검토

텍스트의 문법, 논리, 흐름 오류를 파악합니다. 전체 문서를 다시 쓰지 않고 구체적이고 집중된 수정 제안을 제공합니다.

## 호출 방법

```
/proofread [paste text]
/proofread [upload a document]
```

## 워크플로우

### 1단계: 텍스트 수령

붙여넣기, 업로드된 문서 (DOCX, PDF, 마크다운), 또는 이메일 초안 등 어떤 형태로든 텍스트를 수령하세요.

### 2단계: 분석

**grammar-check** 스킬 적용:

세 가지 카테고리의 문제를 스캔하세요:

**문법**: 철자, 구두점, 주어-동사 일치, 시제 일관성, 관사 사용
**논리**: 모순, 근거 없는 주장, 순환 논리, 불명확한 참조
**흐름**: 어색한 전환, 문장 리듬, 단락 구조, 중복, 가독성

### 3단계: 문제 보고

```
## 교정 보고서

**텍스트 길이**: [단어 수]
**발견된 문제**: [카테고리별 수]

### 문제점

#### 1. [카테고리: 문법/논리/흐름]
- **위치**: "[문제가 있는 인용 텍스트]"
- **문제**: [무엇이 잘못되었는지]
- **수정**: "[수정된 텍스트]"

[각 문제에 대해 반복]

### 요약
- 문법: [X]개 문제
- 논리: [X]개 문제
- 흐름: [X]개 문제
- 전체 품질: [평가]
```

### 4단계: 제안

- "모든 수정 사항을 **적용하여** 정리된 텍스트를 반환할까요?"
- "**특정 섹션**에 더 상세히 집중할까요?"

## 메모

- 수정 제안은 최소한으로 유지하세요 — 필요한 것만 변경하고, 저자의 목소리를 보존하세요
- 비영어권 사용자의 경우, 변경 제안 이유를 특히 명확하게 설명하세요
- 문체 선호도를 과도하게 교정하지 마세요 — 틀린 것과 다른 것은 구별됩니다
- 전문적인 문서의 경우, 톤 일관성과 독자 적합성도 확인하세요
