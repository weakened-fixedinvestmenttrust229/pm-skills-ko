# /tailor-resume -- 이력서-채용공고 최적화

이력서와 목표 채용 공고를 가져와 면접 기회를 극대화하기 위해 경험을 전략적으로 정렬합니다. 키워드 최적화, 글머리 기호 재작성, 갭 분석.

## 호출 방법

```
/tailor-resume [upload resume] Here's the JD: [paste job description]
/tailor-resume [upload both resume and JD as files]
```

## 워크플로우

### 1단계: 두 문서 수령

두 가지 입력이 필요합니다:
- 이력서 (텍스트, PDF, 또는 DOCX)
- 목표 채용 공고 (텍스트, URL, 또는 파일)

하나만 제공되면 나머지를 요청하세요.

### 2단계: 채용 공고 분석

다음을 추출하세요:
- 필수 자격 및 스킬
- 우대 자격
- 핵심 책임
- 업종 및 도메인 신호
- 시니어리티 수준 지표
- 문화 및 팀 신호

### 3단계: 이력서 맞춤화

**review-resume** 스킬 적용:

- **키워드 정렬**: 채용 공고 키워드를 이력서 콘텐츠에 매핑, 누락된 키워드 자연스럽게 추가
- **글머리 기호 재작성**: XYZ+S 공식을 사용하여 채용 공고 관련 성과를 강조하도록 경험 재구성
- **섹션 재정렬**: 가장 관련성 높은 경험 우선
- **요약/목표**: 해당 역할을 직접 다루도록 재작성
- **스킬 섹션**: 채용 공고 요구사항과 정렬

### 4단계: 맞춤화된 이력서 + 분석 생성

```
## 이력서 맞춤화: [회사명]의 [직무명]

### 정렬 점수: [X/10]

### 키워드 갭 분석
| 채용공고 키워드 | 이력서에 있음? | 권장사항 |
|-----------|-----------|---------------|

### 변경 사항
1. **[섹션]**: [변경된 내용과 이유]
2. ...

### 맞춤화된 이력서
[완전히 재작성된 이력서 텍스트]

### 갭 분석
**강한 일치**: [경험이 직접 정렬되는 곳]
**재구성된 일치**: [경험이 맞게 재포지셔닝된 곳]
**갭**: [명확하게 다루지 않는 채용 공고 요구사항 — 제안 포함]

### 커버 레터 핵심 포인트
[남아 있는 갭을 메우는 커버 레터에서 강조할 3~4가지 포인트]
```

맞춤화된 이력서를 마크다운으로 저장.

## 메모

- 경험을 절대 조작하지 마세요 — 진실되게 재구성하고, 만들어 내지 마세요
- 요약/목표 섹션은 지원별 커스터마이제이션의 ROI가 가장 높습니다
- 가능한 경우 채용 공고의 언어와 정확히 일치시키세요 (공고에서 "cross-functional"이라고 하면 "cross-functional" 사용)
- 시니어 역할의 경우 규모와 전략적 임팩트 강조; IC 역할의 경우 실무 실행 강조
