# /privacy-policy -- 개인정보처리방침 생성기

제품에 대한 포괄적인 개인정보처리방침을 초안 작성합니다. 데이터 유형, 관할권, 컴플라이언스 (GDPR, CCPA)를 다루며, 법률 검토가 필요한 조항을 표시합니다.

## 호출 방법

```
/privacy-policy SaaS analytics tool that collects user behavior data — serving US and EU customers
/privacy-policy Mobile app with location data and third-party integrations
```

## 워크플로우

### 1단계: 맥락 수집

다음을 질문하세요:
- 어떤 제품 또는 서비스인가?
- 어떤 데이터가 수집되는가? (개인 정보, 이용 데이터, 쿠키, 위치, 결제)
- 사용자는 어디에 있는가? (적용 가능한 규정 결정: GDPR, CCPA 등)
- 서드파티 데이터 공유가 있는가? (분석, 광고, 연동)
- 데이터 저장: 어디에, 얼마나 오래?
- 연령 제한이 있는가? (COPPA 고려사항)

### 2단계: 방침 초안 작성

**privacy-policy** 스킬 적용:

다음을 포함하는 섹션 생성:
- 수집되는 데이터 및 수집 방법
- 데이터 사용 방법 (목적)
- 처리의 법적 근거 (GDPR)
- 데이터 공유 및 서드파티
- 데이터 보존 및 삭제
- 사용자 권리 (접근, 삭제, 이동성, 거부)
- 쿠키 정책
- 보안 조치
- 아동 프라이버시 (해당 시)
- 국제 이전
- 연락처 정보
- 방침 업데이트 절차

### 3단계: 결과물 전달

```
## Privacy Policy: [제품명]

[전체 방침 텍스트]

### 컴플라이언스 체크리스트
| 규정 | 상태 | 메모 |
|-----------|--------|-------|

### 법률 검토가 필요한 조항
| 조항 | 이유 | 우선순위 |
|--------|-----|----------|

### 구현 체크리스트
- [ ] 쿠키 동의 배너
- [ ] 정보 주체 요청 절차
- [ ] 데이터 처리 기록
- [ ] 처리업체와의 DPA
```

마크다운으로 저장. DOCX 내보내기 제안.

## 메모

- 이것은 템플릿입니다 — 게시 전에 법률 전문가가 검토해야 합니다
- GDPR과 CCPA는 근사치를 낼 수 없는 특정 요구사항이 있습니다 — 전문가 검토가 필수적인 곳을 표시하세요
- 개인정보처리방침은 법률 용어가 아닌 평이한 언어로 작성되어야 합니다
- 연간 업데이트가 아니라 데이터 관행이 변경될 때 방침을 업데이트하세요
