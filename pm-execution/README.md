# pm-execution

실행 및 제품 관리 스킬: PRD, OKR, 로드맵, 스프린트, 프리모템, 이해관계자 맵, 사용자 스토리, 우선순위 프레임워크 등.

## 스킬 (15)

- **brainstorm-okrs** — 회사 목표에 맞춰 팀 수준의 OKR을 브레인스토밍합니다.
- **create-prd** — 요약, 배경, 목표, 시장 세그먼트, 가치 제안, 솔루션 세부 사항, 출시 계획을 포함하는 8개 섹션의 포괄적인 템플릿으로 Product Requirements Document를 작성합니다.
- **dummy-dataset** — 테스트용 현실적인 더미 데이터셋을 커스터마이징 가능한 컬럼, 제약 조건, 출력 형식(CSV, JSON, SQL, Python 스크립트)으로 생성합니다.
- **job-stories** — 'When [상황], I want to [동기], so I can [결과]' 형식으로 상세한 인수 기준과 함께 잡 스토리를 작성합니다.
- **outcome-roadmap** — 아웃풋 중심 로드맵을 아웃컴 중심으로 전환합니다.
- **pre-mortem** — PRD에 대한 프리모템 분석을 실행합니다.
- **prioritization-frameworks** — 9가지 우선순위 프레임워크의 공식, 사용 시기 가이드, 템플릿을 담은 참고 가이드입니다.
- **release-notes** — 티켓, PRD, 또는 변경 로그에서 사용자 대상 릴리스 노트를 생성합니다.
- **retro** — 구조화된 스프린트 회고를 진행합니다.
- **sprint-plan** — 용량 추정, 스토리 선택, 의존성 매핑, 리스크 파악을 통해 스프린트를 계획합니다.
- **stakeholder-map** — 파워/관심도 그리드를 사용해 이해관계자 맵을 구축하고, 각 사분면별 커뮤니케이션 전략을 파악하며, 커뮤니케이션 계획을 수립합니다.
- **summarize-meeting** — 회의 트랜스크립트를 날짜, 참석자, 주제, 요약 포인트, 액션 아이템이 포함된 구조화된 템플릿으로 요약합니다.
- **test-scenarios** — 사용자 스토리로부터 테스트 목표, 시작 조건, 사용자 역할, 단계별 액션, 예상 결과를 포함한 종합적인 테스트 시나리오를 작성합니다.
- **user-stories** — 3C(Card, Conversation, Confirmation)와 INVEST 기준을 따르는 사용자 스토리를 설명, 디자인 링크, 인수 기준과 함께 작성합니다.
- **wwas** — Why-What-Acceptance 형식으로 제품 백로그 아이템을 작성합니다.

## 커맨드 (10)

- `/pm-execution:generate-data` — 테스트용 현실적인 더미 데이터셋 생성 — CSV, JSON, SQL 인서트, 또는 Python 스크립트.
- `/pm-execution:meeting-notes` — 회의 트랜스크립트를 결정 사항, 액션 아이템, 후속 조치가 포함된 구조화된 노트로 요약합니다.
- `/pm-execution:plan-okrs` — 회사 목표에 맞춰 팀 수준의 OKR 브레인스토밍 — 측정 가능한 핵심 결과를 포함한 정성적 목표.
- `/pm-execution:pre-mortem` — PRD, 출시 계획, 또는 기능에 대한 프리모템 리스크 분석 — 문제가 발생하기 전에 무엇이 잘못될 수 있는지 파악합니다.
- `/pm-execution:sprint` — 스프린트 라이프사이클 — 스프린트 계획, 회고 실행, 또는 릴리스 노트 생성.
- `/pm-execution:stakeholder-map` — Power × Interest 그리드에서 이해관계자를 맵핑하고 맞춤형 커뮤니케이션 계획을 작성합니다.
- `/pm-execution:test-scenarios` — 사용자 스토리 또는 기능 스펙에서 종합적인 테스트 시나리오 생성 — 해피 패스, 엣지 케이스, 에러 처리.
- `/pm-execution:transform-roadmap` — 기능 중심 로드맵을 전략적 의도를 전달하는 아웃컴 중심 로드맵으로 전환합니다.
- `/pm-execution:write-prd` — 기능 아이디어나 문제 진술서로부터 포괄적인 Product Requirements Document를 작성합니다.
- `/pm-execution:write-stories` — 기능을 인수 기준이 포함된 사용자 스토리, 잡 스토리, 또는 WWA 형식의 백로그 아이템으로 분해합니다.
