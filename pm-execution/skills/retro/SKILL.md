## 스프린트 회고 진행자

인사이트를 도출하고 실행 가능한 개선 사항을 만들어내는 구조화된 회고를 진행합니다.

### 맥락

**$ARGUMENTS**에 대한 회고를 진행합니다.

사용자가 파일(스프린트 데이터, 속도 차트, 팀 피드백, 또는 이전 회고 메모)을 제공하면 먼저 읽으세요.

### 지시사항

1. **맥락에 맞는 회고 형식 선택** (또는 사용자가 선택하도록):

   **Format A — Start / Stop / Continue**:
   - **Start**: 새로 시작해야 할 것은?
   - **Stop**: 중단해야 할 것은?
   - **Continue**: 계속 유지해야 할 잘 되고 있는 것은?

   **Format B — 4Ls (Liked / Learned / Lacked / Longed For)**:
   - **Liked**: 팀이 즐거웠던 것은?
   - **Learned**: 새롭게 배운 것은?
   - **Lacked**: 부족했던 것은?
   - **Longed For**: 있었으면 했던 것은?

   **Format C — Sailboat**:
   - **Wind (순풍)**: 우리를 앞으로 이끄는 것은?
   - **Anchor (닻)**: 우리를 늦추는 것은?
   - **Rocks (암초)**: 앞에 놓인 위험 요소는?
   - **Island (목적지)**: 우리가 향하는 곳은?

2. **사용자가 날것의 피드백을 제공한 경우** (예: 포스트잇, 설문 응답, Slack 메시지):
   - 유사한 항목을 주제별로 묶기
   - 가장 자주 언급된 주제 파악
   - 감정 패턴 파악 (좌절, 에너지, 혼란)

3. **스프린트 성과 분석**:
   - 스프린트 목표: 달성 여부?
   - 속도 대 커밋 (과도한 커밋? 부족한 커밋?)
   - 발생한 장애물과 해결 방법
   - 협업 패턴 (잘 된 것, 잘 안 된 것)

4. **우선순위 액션 아이템 생성**:

   | 우선순위 | 액션 아이템 | 담당자 | 기한 | 성공 지표 |
   |---|---|---|---|---|
   | 1 | [구체적이고 실행 가능한 개선 사항] | [이름/역할] | [날짜] | [효과 확인 방법] |

   - 2~3개의 액션 아이템으로 제한 (더 많으면 실행이 안 됨)
   - 각 항목은 구체적이고, 담당자를 지정할 수 있으며, 측정 가능해야 함
   - 가능하면 이전 회고 액션 아이템 참조 — 완료되었는가?

5. **회고 요약 작성**:
   ```
   ## Sprint [X] 회고 — [날짜]

   ### 스프린트 성과
   - 목표: [달성 / 부분 달성 / 미달성]
   - 커밋: [X pts] | 완료: [Y pts]

   ### 주요 주제
   1. [주제] — [요약]

   ### 액션 아이템
   1. [액션] — [담당자] — [기한]

   ### 지난 회고에서 이월된 항목
   - [이전 액션] — [상태: 완료 / 진행 중 / 미착수]
   ```

마크다운으로 저장하세요. 톤은 건설적으로 유지하세요 — 목표는 개선이지 책임 추궁이 아닙니다.
