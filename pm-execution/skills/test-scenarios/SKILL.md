# Test Scenarios

유저 스토리를 바탕으로 테스트 목표, 사전 조건, 사용자 역할, 단계별 테스트 액션, 예상 결과를 포함한 종합적인 테스트 시나리오를 작성합니다.

**사용 시기:** QA 테스트 케이스 작성, 테스트 계획 수립, 인수 테스트 시나리오 정의, 유저 스토리 구현 검증 시.

**Arguments:**
- `$PRODUCT`: 제품 또는 시스템 이름
- `$USER_STORY`: 테스트할 유저 스토리 (제목 및 인수 기준)
- `$CONTEXT`: 추가적인 테스트 맥락 또는 제약 조건

## 단계별 프로세스

1. **유저 스토리 검토** - 인수 기준 확인
2. **테스트 목표 정의** - 검증할 구체적인 동작 파악
3. **사전 조건 설정** - 시스템 상태, 데이터 설정, 구성 정보
4. **사용자 역할 식별** - 테스트 액션을 수행할 주체
5. **테스트 단계 작성** - 상호작용을 단계별로 세분화
6. **예상 결과 정의** - 각 단계 후 관찰 가능한 결과
7. **엣지 케이스 고려** - 잘못된 입력값, 경계 조건
8. **상세 테스트 시나리오 출력** - QA 실행 준비 완료 상태로

## 테스트 시나리오 예시

**테스트 시나리오:** 상품 페이지에서 최근 본 상품 표시

**테스트 목표:** '최근 본 상품' 섹션이 올바르게 표시되고 현재 상품이 제외되는지 검증합니다.

**사전 조건:**
- 사용자가 로그인되어 있거나 브라우저 히스토리가 활성화되어 있음
- 현재 세션에서 최소 2개 이상의 상품을 조회한 이력이 있음
- 현재 상품 페이지는 이전에 조회한 상품과 다른 상품임

**사용자 역할:** 온라인 쇼핑객

**테스트 단계:**
1. 임의의 상품 페이지로 이동 → 하단에 이전에 본 상품이 포함된 섹션이 나타나야 함
2. 페이지 하단으로 스크롤 → 상품 카드와 함께 '최근 본 상품' 섹션이 보여야 함
3. 상품 썸네일 확인 → 이미지, 제목, 가격이 올바르게 표시되어야 함
4. 현재 상품 확인 → 현재 상품이 최근 본 상품 목록에 포함되어 있지 않아야 함
5. 상품 카드 클릭 → 해당 상품 페이지로 이동해야 함

**예상 결과:**
- 최근 본 상품 섹션은 이전에 조회한 상품이 최소 1개 이상 있을 때만 표시됨
- 섹션에는 완전한 정보를 갖춘 4~8개의 상품 카드가 표시됨
- 현재 상품은 목록에서 제외됨
- 각 카드에는 "X분/시간 전에 봤습니다" 형식의 타임스탬프가 표시됨
- 카드 클릭 시 올바른 상품 페이지로 이동함
- 성능: 섹션이 2초 이내에 로드됨
