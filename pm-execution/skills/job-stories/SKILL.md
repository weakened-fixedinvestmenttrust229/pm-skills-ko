# Job Stories

'When [상황], I want to [동기], so I can [결과]' 형식으로 잡 스토리를 작성합니다. 사용자 상황과 결과에 초점을 맞춘 상세한 인수 기준이 포함된 스토리를 생성합니다.

**사용 시기:** 잡 스토리 작성, 사용자 상황과 동기 표현, JTBD 방식의 백로그 항목 생성, 또는 역할보다 사용자 맥락에 집중할 때.

**Arguments:**
- `$PRODUCT`: 제품 또는 시스템 이름
- `$FEATURE`: 잡 스토리로 분해할 신규 기능
- `$DESIGN`: 디자인 파일 링크 (Figma, Miro 등)
- `$CONTEXT`: 사용자 상황 또는 잡 시나리오

## 단계별 프로세스

1. **필요를 촉발하는 사용자 상황 파악**
2. **사용자 행동의 근본적인 동기 정의**
3. **사용자가 달성하려는 결과 명확화**
4. **JTBD 프레임워크 적용:** 역할이 아닌 잡(job)에 집중
5. **결과 달성 여부를 검증하는 인수 기준 작성**
6. **관찰 가능하고 측정 가능한 언어 사용**
7. **디자인 목업 또는 프로토타입 링크 연결**
8. **상세 인수 기준이 포함된 잡 스토리 출력**

## 스토리 템플릿

**제목:** [잡의 결과 또는 성과]

**설명:** When [상황], I want to [동기], so I can [결과].

**디자인:** [디자인 파일 링크]

**인수 기준:**
1. [상황이 올바르게 인식됨]
2. [시스템이 원하는 동기를 지원함]
3. [진행 상황 또는 피드백이 표시됨]
4. [결과가 효율적으로 달성됨]
5. [엣지 케이스가 우아하게 처리됨]
6. [연동 및 알림이 작동함]

## 잡 스토리 예시

**제목:** 주간 간식 지출 추적

**설명:** When I'm preparing my weekly allowance for snacks (상황: 주간 간식 비용을 준비할 때), I want to quickly see how much I've spent so far (동기: 지금까지 얼마나 썼는지 빠르게 확인하고 싶음), so I can make sure I don't run out of money before the weekend (결과: 주말 전에 돈이 떨어지지 않도록).

**디자인:** [Figma 링크]

**인수 기준:**
1. '주간 지출 현황' 섹션과 함께 지출 요약 표시
2. 지출 항목 등록 시 실시간 업데이트
3. 주간 예산의 0~100%를 나타내는 진행률 표시줄
4. 눈에 잘 띄는 색상으로 잔여 예산 강조 표시
5. 카테고리별 세부 지출 내역
6. 예산의 80% 도달 시 알림 발송
7. 목요일 저녁 예산의 90% 도달 시 주말 관련 알림
8. 세부 내역으로의 간편한 접근 및 탐색

## 산출물

- 기능에 대한 완전한 잡 스토리 세트
- 각 스토리는 'When...I want...so I can' 형식을 따름
- 결과에 초점을 맞춘 6~8개의 인수 기준
- 사용자 상황과 동기를 강조한 스토리
- 디자인 및 프로토타입으로의 명확한 링크
