# /test-scenarios -- 테스트 시나리오 생성기

사용자 스토리 또는 기능 설명을 QA가 즉시 실행할 수 있는 종합적인 테스트 시나리오로 변환합니다. 해피 패스, 엣지 케이스, 에러 처리, 크로스 브라우저/디바이스 고려 사항을 포함합니다.

## 사용 예시

```
/test-scenarios [사용자 스토리 또는 인수 기준 붙여넣기]
/test-scenarios [PRD 또는 기능 스펙 업로드]
/test-scenarios 사용자가 이메일 링크를 통해 비밀번호를 재설정할 수 있음
```

## 워크플로우

### 1단계: 입력 수락

사용자 스토리, 인수 기준, PRD 섹션, 기능 설명, 또는 예상 동작에 대한 모든 명세를 수락합니다.

### 2단계: 테스트 시나리오 생성

**test-scenarios** 스킬 적용:

각 사용자 스토리 또는 요구사항에 대해 생성합니다:

**해피 패스 시나리오**: 예상되는 사용자 흐름이 올바르게 작동하는 경우
**엣지 케이스**: 경계 조건, 비정상적인 입력, 동시 작업
**에러 시나리오**: 문제가 발생했을 때 어떻게 되는지
**보안 시나리오**: 해당되는 경우 (인증, 권한, 데이터 접근)
**성능 시나리오**: 해당되는 경우 (부하, 타임아웃, 대용량 데이터)

### 3단계: 출력 구조화

```
## 테스트 시나리오: [기능]

**출처**: [사용자 스토리 / PRD / 설명]
**총 시나리오 수**: [개수]
**커버리지**: [해피 패스 / 엣지 케이스 / 에러 / 보안 / 성능]

### 시나리오 1: [제목]
**테스트 대상**: [어떤 스토리 또는 요구사항]
**사전 조건**: [필요한 설정]
**사용자 역할**: [이 작업을 수행하는 사람]

| 단계 | 액션 | 예상 결과 |
|------|--------|----------------|
| 1 | [사용자 액션] | [예상 시스템 응답] |
| 2 | [사용자 액션] | [예상 시스템 응답] |

**사후 조건**: [완료 후 상태]
**우선순위**: [Critical / High / Medium / Low]

---
[각 시나리오마다 반복]

### 커버리지 매트릭스
| 요구사항 | 해피 패스 | 엣지 케이스 | 에러 처리 | 비고 |
|------------|-----------|-----------|---------------|-------|

### 테스트 데이터 요구사항
[이 시나리오를 실행하는 데 필요한 테스트 데이터]
```

마크다운으로 저장합니다.

### 4단계: 다음 단계 제안

- "이 시나리오를 위한 **테스트 데이터를 생성**해 드릴까요?"
- "특정 시나리오에 **더 많은 엣지 케이스를 추가**해 드릴까요?"
- "이 시나리오가 테스트하는 **사용자 스토리를 작성**해 드릴까요?"

## 참고사항

- 먼저 해피 패스를 다루고 엣지 케이스를 층층이 추가하세요 — 경계를 테스트하기 전에 기본 흐름이 작동하는지 확인
- 원래 스토리의 모든 인수 기준은 최소 하나의 테스트 시나리오에 매핑되어야 합니다
- 긍정 테스트(작동함)와 부정 테스트(정상적으로 실패함) 모두 포함하세요
- API의 경우 속도 제한, 타임아웃, 잘못된 요청, 인증 실패에 대한 시나리오를 포함하세요
- 특정 테스트 환경이나 서드파티 서비스 모킹이 필요한 시나리오를 표시하세요
