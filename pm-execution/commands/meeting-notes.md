# /meeting-notes -- 회의 요약

원시 회의 트랜스크립트나 간략한 노트를 결정 사항과 액션 아이템이 명확히 담긴 구조화된 회의록으로 변환합니다.

## 사용 예시

```
/meeting-notes [트랜스크립트 붙여넣기]
/meeting-notes [트랜스크립트 파일, 음성 요약, 또는 노트 업로드]
```

## 워크플로우

### 1단계: 트랜스크립트 수락

다음 형식으로 수락합니다:
- 전체 트랜스크립트 (Otter, Fireflies, Google Meet, Zoom 등에서 생성)
- 회의 중 작성한 간략한 노트
- 트랜스크립션 도구의 음성 요약 또는 회의 요약
- 여러 입력 (예: 트랜스크립트 + 사용자 자신의 노트)

입력 내용이 부족할 경우, 가능한 내용으로 작업하고 누락된 부분을 표시합니다.

### 2단계: 추출 및 구조화

**summarize-meeting** 스킬 적용:

다음을 파악하기 위해 내용을 분석합니다:
- **참석자**: 참석한 사람 (소개, 발화자 레이블, 또는 언급에서)
- **논의된 주제**: 주요 안건 항목 또는 대화 주제
- **결정 사항**: 명시적인 합의 또는 도출된 결론
- **액션 아이템**: 할당된 작업, 가능한 경우 담당자와 마감일 포함
- **미해결 질문**: 후속 조치가 필요한 미결 항목
- **주요 인용문**: 그대로 보존할 가치가 있는 중요한 발언
- **맥락**: 회의 유형, 프로젝트, 배경

### 3단계: 회의 요약 생성

```
## 회의 요약

**날짜**: [알 경우 날짜]
**참석자**: [이름/역할]
**회의 유형**: [스탠드업, 플래닝, 리뷰, 1:1, 이해관계자, 등]
**주제**: [주요 주제]

### 요약
[논의 및 결론에 대한 3-5문장 개요]

### 주요 결정 사항
1. **[결정 사항]** — [맥락 및 근거]
2. ...

### 액션 아이템
| # | 액션 | 담당자 | 마감일 | 상태 |
|---|--------|-------|----------|--------|

### 토론 하이라이트
**[주제 1]**: [주요 포인트, 다양한 관점, 결론]
**[주제 2]**: [주요 포인트, 다양한 관점, 결론]

### 미결 질문
- [질문] — [담당자/팀]의 의견 필요

### 다음 단계
- [다음에 할 일]
- 다음 회의: [언급된 경우]
```

마크다운으로 저장합니다.

### 4단계: 후속 제안

- "참석자에게 **이 노트를 이메일로 전송**해 드릴까요?"
- "액션 아이템에서 **티켓을 생성**해 드릴까요?"
- "내려진 결정을 바탕으로 **이해관계자 업데이트를 작성**해 드릴까요?"

## 참고사항

- 결정 사항이 가장 중요한 산출물입니다 — 모든 결정 사항이 명확히 기록되었는지 확인하세요
- 담당자 없는 액션 아이템은 무용지물입니다 — 담당자가 언급되지 않았다면 표시하세요
- 요약은 간결하게 유지하세요 — 회의에 참석하지 않은 사람도 30초 안에 핵심을 파악할 수 있어야 합니다
- 트랜스크립트가 매우 긴 경우 (60분 이상 회의), 전체 요약 전에 TL;DR을 제공하세요
- "논의됨"과 "결정됨"을 구분하세요 — 많은 주제가 결론 없이 탐색되기만 합니다
