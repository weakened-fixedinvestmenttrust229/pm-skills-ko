# /generate-data -- 테스트 데이터 생성기

개발, 테스트, 데모, 또는 프로토타이핑을 위한 현실적인 더미 데이터셋을 생성합니다. 원하는 형식의 바로 사용 가능한 파일로 출력합니다.

## 사용 예시

```
/generate-data 이름, 이메일, 플랜 등급, 가입 날짜, 활동 점수가 포함된 사용자 1000명
/generate-data 전자상거래 주문 데이터셋: 상품, 고객, 타임스탬프, 금액
/generate-data 이 스키마에 맞는 샘플 데이터: [테이블 정의 붙여넣기]
```

## 워크플로우

### 1단계: 데이터셋 정의

다음을 파악합니다:
- 어떤 엔티티인가요? (사용자, 주문, 이벤트, 상품 등)
- 어떤 컬럼이 필요한가요? (데이터 타입 및 제약 조건 포함)
- 몇 개의 행이 필요한가요?
- 테이블 간 관계가 있나요?
- 특정 분포가 있나요? (예: "80%는 무료 플랜이어야 함")
- 현실적인 제약 조건이 있나요? (이메일은 고유해야 하고, 날짜는 시간순이어야 함)

### 2단계: 데이터 생성

**dummy-dataset** 스킬 적용:

- 데이터셋을 생성하는 Python 스크립트 작성
- 현실적으로 보이는 데이터 사용 (무작위 문자열 아님): 적절한 이름, 유효한 이메일 형식, 실제처럼 보이는 날짜
- 제약 조건 준수: 고유 ID, 외래 키 관계, 시간순 정렬
- 지정된 분포 적용
- 스크립트 실행 후 출력 파일 생성

### 3단계: 결과물 전달

요청된 형식으로 출력 (또는 사용자에게 확인):
- **CSV**: 가장 일반적이며 어디서나 사용 가능
- **JSON**: API 테스트 또는 프런트엔드 개발용
- **SQL INSERT**: 테스트 데이터베이스 채우기용
- **Python 스크립트**: 재현 가능한 생성용 (사용자가 수정하여 재실행 가능)

```
## 생성된 데이터셋: [설명]

**행 수**: [개수]
**컬럼**: [목록]
**형식**: [CSV / JSON / SQL / Python]

### 스키마
| 컬럼 | 타입 | 제약 조건 | 분포 |
|--------|------|-----------|-------------|

### 샘플 (첫 5행)
[데이터 미리보기]

### 파일
- [데이터 파일]
- [생성기 스크립트, 해당되는 경우]
```

데이터 파일과 생성기 스크립트를 사용자의 작업 공간에 저장합니다.

### 4단계: 후속 제안

- "**컬럼 추가** 또는 **데이터셋 크기 증가**를 원하시나요?"
- "**관련 테이블**을 작성해 드릴까요? (예: 이 사용자들의 주문)"
- "이 데이터를 사용하는 **테스트 시나리오를 작성**해 드릴까요?"
- "이 데이터셋을 분석하는 **SQL 쿼리를 작성**해 드릴까요?"

## 참고사항

- 사용자가 다른 파라미터로 재생성할 수 있도록 항상 생성기 스크립트를 제공하세요
- 데모 데이터셋의 경우 데이터가 스토리를 담도록 만드세요 (예: 계절적 트렌드, 리텐션 문제, 파워 유저 세그먼트)
- 현실적인 카디널리티를 고려하세요: 사용자 1000명이 1000개의 고유한 도시를 갖지는 않습니다
- 금융 데이터의 경우 현실적인 가격 분포를 사용하세요 — 균일 분포 사용 금지
- 실제 개인 정보를 포함하지 마세요 — 모든 이름, 이메일, 식별자는 가상이어야 합니다
