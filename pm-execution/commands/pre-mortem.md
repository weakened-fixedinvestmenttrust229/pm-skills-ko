# /pre-mortem -- 출시 전 리스크 분석

출시가 실패했다고 가정하고 원인을 역추적합니다. 이 커맨드는 Tigers/Paper Tigers/Elephants 프레임워크를 적용해 실제 리스크를 표면화하고 완화 계획을 수립합니다.

## 사용 예시

```
/pre-mortem [PRD, 출시 계획, 또는 기능 스펙 붙여넣기 또는 업로드]
/pre-mortem 다음 달에 셀프서비스 빌링 포털을 출시할 예정입니다
```

## 워크플로우

### 1단계: 계획 수락

PRD, 기능 스펙, 출시 계획, 프로젝트 브리프, 또는 구두 설명 어떤 형식으로든 수락합니다. 상세할수록 리스크 분석이 더 정교해집니다.

### 2단계: 리스크 파악

**pre-mortem** 스킬 적용:

제품이 출시되어 실패했다고 가정합니다. 다음 카테고리에서 리스크를 생성합니다:
- **기술적**: 성능, 확장성, 통합 실패, 데이터 문제
- **사용자**: 도입 장벽, 사용성 문제, 충족되지 않은 기대
- **비즈니스**: 매출 영향, 경쟁사 대응, 시장 타이밍
- **운영적**: 지원 부하, 문서화 공백, 교육 필요성
- **의존성**: 서드파티 서비스, 팀 간 핸드오프, 규제

### 3단계: 리스크 분류

각 리스크를 분류합니다:

**Tigers** — 실패를 야기할 수 있는 실제적이고 실질적인 리스크
- 심각도 평가: 출시 차단 / 빠른 후속 조치 / 추적
- 출시 차단 Tigers: 즉각적인 완화 조치 필요
- 빠른 후속 조치 Tigers: 출시 후 첫 스프린트 내 해결 계획 수립
- 추적 Tigers: 모니터링하되 출시 지연 불필요

**Paper Tigers** — 무섭게 느껴지지만 과장된 리스크
- 해당 우려가 관리 가능한 이유 설명
- 이것이 실제 Tiger가 되려면 무엇이 바뀌어야 하는지 기록

**Elephants** — 팀이 알고 있지만 논의를 피하는 미언급 리스크
- 정치적, 조직적, 불편한 리스크를 표면화
- 제안된 대화 시작 방법과 함께 건설적으로 구성

### 4단계: 프리모템 보고서 생성

```
## 프리모템: [기능/출시]

**날짜**: [오늘]
**상태**: [초안 / 검토됨]

### 리스크 요약
- **Tigers**: [개수] ([출시 차단], [빠른 후속 조치], [추적])
- **Paper Tigers**: [개수]
- **Elephants**: [개수]

### 출시 차단 Tigers
| # | 리스크 | 발생 가능성 | 영향 | 완화 방안 | 담당자 | 마감일 |
|---|------|-----------|--------|-----------|-------|----------|

### 빠른 후속 조치 Tigers
| # | 리스크 | 발생 가능성 | 영향 | 대응 계획 | 담당자 |
|---|------|-----------|--------|-----------------|-------|

### 추적 Tigers
[출시 후 모니터링할 리스크와 트리거 조건]

### Paper Tigers
[크게 보이지만 관리 가능한 우려 사항 — 근거 포함]

### 방 안의 코끼리 (Elephants)
[팀이 논의해야 할 불편한 사실]

### Go/No-Go 체크리스트
- [ ] 모든 출시 차단 Tigers 완화됨
- [ ] 빠른 후속 조치 계획 문서화 및 할당됨
- [ ] 추적 Tigers에 대한 모니터링 체계 구축됨
- [ ] 롤백 계획 정의됨
- [ ] 지원 팀 브리핑 완료됨
```

마크다운으로 저장합니다.

### 5단계: 다음 단계 제안

- "리스크 완화 방안으로 **PRD를 업데이트**해 드릴까요?"
- "가장 위험한 영역에 대한 **테스트 시나리오를 작성**해 드릴까요?"
- "이 발견 사항에서 **출시 체크리스트를 작성**해 드릴까요?"

## 참고사항

- 가장 좋은 프리모템은 계획이 80% 완성되었을 때 실시합니다 — 방향을 바꿀 수 있을 만큼 충분히 이르고, 실질적인 내용이 있을 만큼 충분히 늦은 시점
- 뻔한 리스크를 넘어서 밀어붙이세요 — 가장 위험한 리스크는 아무도 언급하지 않는 것들입니다
- Elephants가 가장 높은 가치의 산출물입니다 — 팀이 논의를 피하는 것을 표면화하는 것
- 각 Tiger에 대해 완화 방안은 구체적이고 할당 가능해야 합니다, "조심하기"가 아닌
- 프리모템에서 너무 많은 출시 차단 Tigers가 발견된다면, 출시를 지연하거나 단계적으로 진행할 것을 권장하세요
