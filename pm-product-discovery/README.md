# pm-product-discovery

PM을 위한 프로덕트 디스커버리 스킬: 아이디에이션, 실험 설계, 가정 검증, 기능 우선순위 결정, 고객 인터뷰 종합.

## 스킬 (13)

- **analyze-feature-requests** — 기능 요청 목록을 테마, 전략적 정렬, 영향, 노력, 위험도 기준으로 분석하고 우선순위를 결정합니다.
- **brainstorm-experiments-existing** — 기존 제품의 가정을 검증하는 실험을 설계합니다.
- **brainstorm-experiments-new** — 신규 제품을 위한 린 스타트업 실험(프리토타입)을 설계합니다.
- **brainstorm-ideas-existing** — PM, 디자이너, 엔지니어의 다각적 시각을 활용해 기존 제품의 아이디어를 도출합니다.
- **brainstorm-ideas-new** — 초기 디스커버리 단계에서 신규 제품의 기능 아이디어를 도출합니다.
- **identify-assumptions-existing** — 기존 제품의 기능 아이디어에 대한 가치, 사용성, 실행 가능성, 기술 타당성 측면의 위험 가정을 식별합니다.
- **identify-assumptions-new** — Go-to-Market, 전략, 팀을 포함한 8가지 위험 카테고리에 걸쳐 신규 제품 아이디어의 위험 가정을 식별합니다.
- **interview-script** — JTBD 탐색 질문, 워밍업, 핵심 탐색, 마무리 섹션을 포함한 구조화된 고객 인터뷰 스크립트를 작성합니다.
- **metrics-dashboard** — 핵심 지표, 데이터 소스, 시각화 유형, 알림 임계값을 포함한 프로덕트 지표 대시보드를 정의하고 설계합니다.
- **opportunity-solution-tree** — 프로덕트 디스커버리 구조화를 위한 기회-솔루션 트리(OST)를 구축합니다 — 원하는 결과를 기회, 솔루션, 실험과 연결합니다.
- **prioritize-assumptions** — 영향 × 위험 매트릭스를 활용해 가정의 우선순위를 정하고 각 가정에 맞는 실험을 제안합니다.
- **prioritize-features** — 영향, 노력, 위험, 전략적 정렬을 기반으로 기능 아이디어 백로그의 우선순위를 결정합니다.
- **summarize-interview** — 고객 인터뷰 대화록을 JTBD, 만족 신호, 액션 아이템이 포함된 구조화된 템플릿으로 요약합니다.

## 커맨드 (5)

- `/pm-product-discovery:brainstorm` — PM, 디자이너, 엔지니어 시각에서 기존 또는 신규 제품의 아이디어나 실험을 도출합니다.
- `/pm-product-discovery:discover` — 아이디에이션부터 가정 매핑, 실험 설계까지 전체 프로덕트 디스커버리 사이클을 실행합니다.
- `/pm-product-discovery:interview` — 고객 인터뷰 스크립트를 준비하거나 인터뷰 대화록을 구조화된 인사이트로 요약합니다.
- `/pm-product-discovery:setup-metrics` — 북극성 지표, 입력 지표, 건강 지표, 알림 임계값을 포함한 프로덕트 지표 대시보드를 설계합니다.
- `/pm-product-discovery:triage-requests` — 고객이나 이해관계자로부터 받은 기능 요청 묶음을 분석, 분류, 우선순위화합니다.
