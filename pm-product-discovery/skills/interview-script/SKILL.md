## 고객 인터뷰 스크립트

의견이 아닌 실제 인사이트를 발굴하는 구조화된 인터뷰 스크립트를 작성합니다. "엄마 테스트(The Mom Test)" 원칙을 따릅니다 — 내 아이디어가 아닌 그들의 삶에 대해 질문합니다.

### 도메인 맥락

고객 인터뷰는 지속적 디스커버리의 **1단계(탐색)**에서 하나의 소스입니다. 다른 소스로는 이해관계자 인터뷰, 사용 분석, 데이터 분석, 설문조사, 시장 트렌드, SEO/SEM 분석이 있습니다. PM은 "프록시 없이" 사용자, 이해관계자, 엔지니어, 디자이너에게 직접 접근해야 합니다. **프로덕트 트리오** (PM + 디자이너 + 엔지니어 — Teresa Torres)는 PM 혼자가 아니라 함께 디스커버리를 진행해야 합니다.

### 맥락

**$ARGUMENTS**에 대한 리서치를 위한 고객 인터뷰 스크립트를 준비합니다.

사용자가 파일(페르소나, 가설 목록, 제품 브리프, 또는 이전 인터뷰 메모)을 제공하면 먼저 읽습니다.

### 지시 사항

1. **리서치 목표 명확화**:
   - 팀이 답을 얻어야 하는 구체적인 질문은 무엇인가요?
   - 이 리서치가 어떤 의사 결정에 영향을 미칠까요?
   - 어떤 가정을 검증해야 하나요?

2. **다음 섹션으로 인터뷰 스크립트를 작성합니다**:

   ### 오프닝 (2-3분)
   - 자기 소개와 목적 설명 (판매가 아닌 학습)
   - 기대 설정: "맞고 틀린 답이 없습니다. 여러분의 경험에서 배우러 왔습니다."
   - 녹음 허가 요청 (해당하는 경우)
   - 가용 시간 확인

   ### 워밍업: 맥락 및 배경 (5분)
   - "역할과 일반적인 하루/주가 어떤지 말씀해 주세요."
   - "[제품 영역과 관련된 활동]을 얼마나 오래 하셨나요?"
   - 목표: 관계 형성 및 맥락 이해

   ### 핵심 탐색: 수행해야 할 작업(JTBD) (15-20분)

   **현재 상황과 행동** (과거형, 구체적인 사례):
   - "[탐색하는 것]을 마지막으로 했을 때를 설명해 주세요. 어떻게 됐나요?"
   - "어떤 도구나 방법을 사용하셨나요?"
   - "얼마나 걸렸나요? 누가 관련됐나요?"

   **불편한 점과 좌절감** (관찰하되 유도하지 않음):
   - "가장 힘든 부분은 무엇이었나요?"
   - "마법 지팡이가 있다면 무엇을 바꾸시겠어요?"
   - "이를 해결하기 위해 무엇을 시도해 보셨나요? 어떻게 됐나요?"

   **원하는 결과** (그들의 말로, 당신의 말이 아닌):
   - "이 영역에서 '잘 된다'는 것은 어떤 모습인가요?"
   - "이것이 잘 작동하는지 어떻게 알 수 있을까요?"

   **구매 의향 / 우선순위** (실질적인 약속):
   - "현재 이것에 얼마나 많은 시간/돈을 쓰고 있나요?"
   - "더 나은 솔루션을 찾아보신 적 있나요? 무엇을 발견하셨나요?"
   - "이것이 해결된다면 무엇을 포기할 의향이 있으신가요?"

   ### 탐색 기법
   흥미로운 실마리에 도달했을 때 사용합니다:
   - **"그것에 대해 더 말씀해 주세요"** — 어떤 주제든 열어줍니다
   - **"왜요?"** (부드럽게, 2-3번) — 근본 원인에 도달합니다
   - **"구체적인 예를 들어주실 수 있나요?"** — 의견에서 사실로 이동합니다
   - **"그 다음엔 어떻게 됐나요?"** — 이야기를 따라갑니다
   - **"그때 어떤 기분이었나요?"** — 감정적 강도를 포착합니다

   ### 엄마 테스트 규칙
   - **그들의 삶**에 대해 질문합니다, 당신의 아이디어가 아닌
   - **과거**에 대해 질문합니다, 미래가 아닌 ("X를 사용하실 건가요?"는 쓸모없습니다)
   - **덜 말하고, 더 듣습니다** — 80/20 분할을 목표로 합니다
   - 인터뷰 중에 **절대 홍보하지 않습니다**
   - **강한 감정**을 찾습니다 — 실제 고통이나 즐거움을 나타냅니다
   - **칭찬은 소음입니다** — "멋있어 보여요!"는 아무것도 알려주지 않습니다

   ### 마무리 (3-5분)
   - "제가 묻지 않은 것 중 중요하다고 생각하시는 것이 있나요?"
   - "이것에 대해 이야기해야 할 다른 사람을 추천해 주실 수 있나요?"
   - 시간을 내주신 것에 감사드립니다
   - 다음 단계 공유 (있는 경우)

3. **스크립트 맞춤화**: 특정 제품 영역, 페르소나, 리서치 목표에 맞게 질문을 조정합니다. 인터뷰 가용 시간에 따라 섹션을 추가하거나 제거합니다.

4. **메모 작성 템플릿 포함**:
   ```
   참가자: [이름 / ID]
   날짜: [날짜]
   주요 작업(JTBD): [달성하려는 것]
   현재 솔루션: [현재 사용하는 것]
   가장 큰 불편함: [최우선 좌절]
   원하는 결과: [성공의 모습]
   구매 의향: [현재 / 투자할 의향]
   놀라운 발견: [예상치 못한 것]
   후속 조치: [다음 단계]
   ```

마크다운으로 저장합니다. 스크립트와 메모 작성 템플릿을 모두 포함합니다.
