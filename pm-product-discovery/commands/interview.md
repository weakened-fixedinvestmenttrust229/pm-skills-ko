# /interview -- 고객 인터뷰 준비 및 요약

두 가지 모드: **prep**은 고객과 대화하기 전에 구조화된 인터뷰 스크립트를 작성하고, **summarize**는 인터뷰 후 인사이트를 추출합니다.

## 실행 방법

```
/interview prep Onboarding experience for enterprise users
/interview summarize [paste transcript or upload file]
/interview                    # 어떤 모드가 필요한지 질문합니다
```

## 모드

---

### 준비(Prep) 모드

리서치 질문에 맞는 구조화된 인터뷰 스크립트를 작성합니다.

#### 워크플로우

**1단계: 리서치 목표 파악**

사용자에게 질문합니다:
- 무엇을 알고 싶으신가요? (구체적인 리서치 질문)
- 누구를 인터뷰하나요? (세그먼트, 역할, 제품과의 관계)
- 얼마나 시간이 있나요? (15분, 30분, 60분)
- 이 리서치가 어떤 의사 결정에 영향을 미칠까요?

**2단계: 인터뷰 스크립트 생성**

**interview-script** 스킬을 적용합니다:

- "엄마 테스트(The Mom Test)" 원칙을 따릅니다 — 내 아이디어가 아닌 그들의 삶에 대해 질문합니다
- 유도 질문 없음, 홍보 없음, 과거 행동과 실제 상황에 집중
- 다음 섹션으로 스크립트를 구성합니다:

```
## 인터뷰 스크립트: [리서치 주제]

**리서치 질문**: [알고 싶은 것]
**대상 참가자**: [누구]
**소요 시간**: [X]분

### 워밍업 (3-5분)
[관계 형성 질문, 역할/맥락 파악]

### 핵심 탐색 (15-40분)
[JTBD 탐색, 과거 행동, 현재 워크플로우, 불편한 점]
- 각 질문마다: 질문 + 질문하는 이유 + 후속 질문

### 특정 주제 (5-10분)
[필요한 경우 특정 기능이나 개념에 대한 타겟 질문]

### 마무리 (3-5분)
[개방형 마무리, 추천 요청, 다음 단계]

### 메모 작성 템플릿
[인터뷰 중 인사이트를 기록하기 위한 사전 형식화된 템플릿]

### 주의해야 할 신호
[대화가 잘못 진행되거나 참가자가 솔직하지 않고 예의 바르게 대답하는 징후]
```

**3단계: 맞춤화 및 검토**

- 시간 슬롯에 맞게 질문 수 조정
- 사용자가 테스트하려는 특정 가설에 대한 탐색 질문 추가
- 증인을 유도할 수 있는 질문 표시
- 인쇄 가능한 버전 제공 (작업 공간에 마크다운 파일로 저장)

---

### 요약(Summarize) 모드

인터뷰 대화록을 구조화된 실행 가능한 인사이트로 변환합니다.

#### 워크플로우

**1단계: 대화록 수령**

어떤 형식으로든 수령합니다:
- **붙여넣기 텍스트**: 원본 대화록 또는 메모
- **업로드 파일**: 문서, 텍스트 파일, 또는 회의 메모 내보내기
- **음성 요약**: 사용자가 말한 내용을 설명하는 경우 (전체 대화록이 아닌 경우)

입력이 전체 대화록이 아닌 거친 메모인 경우, 가능한 내용으로 작업하고 한계를 명시합니다.

**2단계: 추출 및 구조화**

**summarize-interview** 스킬을 적용합니다:

대화록을 파싱하여 다음을 식별합니다:
- **참가자 프로필**: 역할, 경험 수준, 세그먼트, 맥락
- **수행해야 할 작업(JTBD)**: 참가자가 달성하려는 것
- **현재 워크플로우**: 현재 문제를 해결하는 방법
- **불편한 점**: 불만, 우회 방법, 시간 낭비
- **만족 신호**: 잘 작동하는 것, 즐거운 순간
- **인용문**: 핵심 인사이트를 담은 직접 인용 (가능하면 타임스탬프 포함)
- **놀라운 발견**: 예상치 못했거나 가정에 반하는 내용
- **기능 반응**: 특정 기능/개념이 논의된 경우, 반응 기록

**3단계: 인터뷰 요약 생성**

```
## 인터뷰 요약

**참가자**: [익명화된 프로필 — 역할, 세그먼트, 경험]
**날짜**: [알고 있는 경우]
**소요 시간**: [알고 있는 경우]
**인터뷰어**: [알고 있는 경우]

### 핵심 인사이트
1. **[인사이트]** — [뒷받침하는 증거/인용]
2. **[인사이트]** — [뒷받침하는 증거/인용]
3. ...

### 수행해야 할 작업(JTBD)
- **주요 JTBD**: [내가...할 때, 나는...하고 싶다, 그래서...할 수 있다]
- **관련 JTBD**: [추가 작업들]

### 현재 워크플로우
[참가자가 현재 문제를 해결하는 방법, 단계별]

### 불편한 점
| 불편한 점 | 심각도 | 인용 |
|-----------|----------|-------|

### 만족 신호
| 잘 작동하는 것 | 이유 | 인용 |
|-----------|-----|-------|

### 주요 인용문
> "[인용]" — [주제]에 대해

### 검증/무효화된 가정
| 가정 | 상태 | 증거 |
|-----------|--------|----------|

### 액션 아이템
- [ ] [이 인터뷰로부터의 후속 조치]
- [ ] [추가 탐색이 필요한 리서치 질문]

### 원본 메모
[필요한 경우 주석이 달린 핵심 섹션 포함]
```

요약을 마크다운 파일로 저장합니다.

**4단계: 더 넓은 리서치와 연결**

다음을 제안합니다:
- "이것을 수행한 **다른 인터뷰 요약과 비교**할까요?"
- "이 참가자가 말한 것을 바탕으로 **가정을 업데이트**할까요?"
- "여러 인터뷰에서 **페르소나를 추출**할까요?"

## 참고 사항

- 준비 모드에서는 항상 "질문하는 이유" 주석을 포함합니다 — 인터뷰어가 트랙을 유지하는 데 도움이 됩니다
- 요약 모드에서는 참가자가 *말한* 것과 *실제로 한* 것을 구분합니다 (행동 > 진술)
- 같은 인터뷰 내 모순을 표시합니다 (한 가지를 말하고 다른 것을 한다고 설명)
- 대화록에 경쟁 제품이 언급되면 경쟁 정보를 수집합니다
- 요약 모드에서 여러 대화록이 제공되면 참가자 간 패턴을 종합하여 교차 분석합니다
