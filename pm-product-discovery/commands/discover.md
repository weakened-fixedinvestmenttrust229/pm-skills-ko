# /discover -- 전체 디스커버리 사이클

발산적 사고에서 집중된 검증으로 이어지는 구조화된 프로덕트 디스커버리 프로세스를 실행합니다. 이 커맨드는 여러 스킬을 하나의 엔드-투-엔드 워크플로우로 연결합니다.

## 실행 방법

```
/discover Smart notification system for our project management tool
/discover New product: AI writing assistant for non-native speakers
/discover                    # 무엇을 발견하려는지 질문합니다
```

## 워크플로우

### 1단계: 디스커버리 맥락 파악

다음 중 어느 경우인지 확인합니다:
- **기존 제품** — 실제 사용자가 있는 알려진 제품에 대한 지속적 디스커버리
- **신규 제품** — 검증된 수요 없이 개념에 대한 초기 디스커버리

사용자에게 질문합니다:
- 무엇을 탐색하고 있나요? (제품 아이디어, 기능 영역, 기회 공간)
- 이미 알고 있는 것은 무엇인가요? (이전 리서치, 고객 피드백, 데이터)
- 이 디스커버리가 어떤 의사 결정에 영향을 미칠까요? (빌드/폐기, 우선순위 결정, 피벗)

업로드된 파일(리서치, PRD, 대화록, 데이터), 링크, 대화를 통해 맥락을 받아들입니다.

### 2단계: 아이디어 브레인스토밍 (발산 단계)

**brainstorm-ideas-existing** 또는 **brainstorm-ideas-new** 스킬을 적용합니다:

- PM, 디자이너, 엔지니어 시각에서 아이디어 생성
- 간략한 근거와 함께 상위 10개 아이디어 제시
- 사용자에게 계속 진행할 3-5개 아이디어를 선택하거나 모두 수용하도록 요청

**체크포인트**: "10개의 아이디어가 있습니다. 어떤 것을 검증해볼까요? 3-5개를 선택하거나 모두 진행할 수 있습니다."

### 3단계: 가정 식별 (비판적 사고 단계)

선택된 각 아이디어에 대해 **identify-assumptions-existing** 또는 **identify-assumptions-new** 스킬을 적용합니다:

- 위험 카테고리에 걸쳐 가정 발굴:
  - **가치(Value)**: 사용자가 이것을 원할까요?
  - **사용성(Usability)**: 사용자가 사용 방법을 이해할 수 있을까요?
  - **기술 타당성(Feasibility)**: 우리가 구축할 수 있을까요?
  - **실행 가능성(Viability)**: 비즈니스 케이스가 성립하나요?
  - **Go-to-Market** (신규 제품만): 사용자에게 도달하고 전환시킬 수 있나요?
- 악마의 변호인 역할의 다각적 분석 활용
- 모든 아이디어에 걸친 가정 마스터 목록 작성

### 4단계: 가정 우선순위 결정 (집중 단계)

**prioritize-assumptions** 스킬을 적용합니다:

- 영향 × 위험 매트릭스에 가정 매핑
- "믿음의 도약(leap of faith)" 가정 식별 — 높은 영향, 높은 불확실성
- 테스트 우선순위에 따라 가정 순위 결정
- 함께 테스트할 수 있는 관련 가정 그룹화

**체크포인트**: "가장 위험한 가정들이 있습니다. 어떤 것을 먼저 검증하는 것이 가장 중요하다고 느끼시나요?"

### 5단계: 실험 설계 (검증 단계)

최우선 가정에 대해 **brainstorm-experiments-existing** 또는 **brainstorm-experiments-new** 스킬을 적용합니다:

- 각 핵심 가정에 대해 1-2개의 실험 설계
- 기존 제품의 경우: A/B 테스트, 가짜 문, 프로토타입, 사용자 테스트, 데이터 분석
- 신규 제품의 경우: XYZ 가설, 프리토타입, 랜딩 페이지, 컨시어지 MVP
- 각각에 대한 성공 기준, 타임라인, 노력 포함
- 의존성과 노력을 고려한 실험 순서 배열

### 6단계: 디스커버리 계획 작성

모든 내용을 디스커버리 계획 문서로 취합합니다:

```
## 디스커버리 계획: [주제]

**날짜**: [오늘]
**제품 단계**: [기존/신규]
**디스커버리 질문**: [우리가 배우려는 것]

### 탐색된 아이디어
[간략한 설명과 함께 브레인스토밍된 아이디어 요약]

### 검증을 위해 선택된 아이디어
[근거와 함께 계속 진행할 3-5개 아이디어]

### 핵심 가정
| # | 가정 | 카테고리 | 영향 | 불확실성 | 우선순위 |
|---|-----------|----------|--------|-------------|----------|

### 검증 실험
| # | 검증하는 가정 | 방법 | 성공 기준 | 노력 | 타임라인 |
|---|-----------------|--------|-----------------|--------|----------|

### 실험 세부 내용
[각 실험에 대해: 가설, 설정, 측정, 의사 결정 기준]

### 디스커버리 타임라인
1주차: [실험들]
2주차: [실험들]
3주차: [분석 및 의사 결정]

### 의사 결정 프레임워크
- [실험]이 성공하면 → [다음 단계]로 진행
- [실험]이 실패하면 → [피벗/폐기/추가 조사]
```

계획을 사용자의 작업 공간에 마크다운 파일로 저장합니다.

### 7단계: 다음 단계 제안

- "최상위 아이디어의 **PRD를 작성**할까요?"
- "실험을 보완할 **인터뷰 스크립트를 설계**할까요?"
- "실험을 추적하기 위한 **지표를 설정**할까요?"
- "MVP를 위한 **노력을 추정하고 사용자 스토리를 작성**할까요?"

## 참고 사항

- 이는 15-30분의 구조화된 워크플로우입니다 — 사전에 사용자에게 알려줍니다
- 각 체크포인트에서 사용자는 방향을 바꾸거나, 건너뛰거나, 더 깊이 파고들 수 있습니다
- 사용자에게 리서치 데이터가 있다면 브레인스토밍 전에 인사이트를 추출합니다
- 디스커버리 계획은 살아있는 문서여야 합니다 — 실험이 진행됨에 따라 업데이트를 제안합니다
- 신규 제품의 경우, 기술 타당성 검증 전에 바람직성 검증을 강조합니다
- 기존 제품의 경우, 가정을 뒷받침할 수 있는 사용 데이터가 있는지 확인합니다
