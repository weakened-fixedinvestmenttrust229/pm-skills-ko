# /analyze-test -- A/B 테스트 분석

통계적 엄밀성으로 실험 결과를 평가하고 명확한 제품 결정으로 전환합니다: 출시, 연장, 또는 중단.

## 실행 예시

```
/analyze-test Control: 4.2% conversion (n=5000), Variant: 4.8% conversion (n=5100)
/analyze-test [테스트 결과 CSV 업로드]
/analyze-test [실험 플랫폼 스크린샷]
```

## 워크플로

### Step 1: 테스트 데이터 수집

다음 형식을 모두 허용합니다:
- 요약 통계 (전환율, 변형별 샘플 크기)
- 원시 이벤트 데이터 (user_id, variant, converted, timestamp가 포함된 CSV)
- 실험 플랫폼 스크린샷 (Optimizely, LaunchDarkly 등)
- 실험 및 결과 설명

### Step 2: 테스트 설계 검증

결과를 분석하기 전에 다음을 확인합니다:
- 샘플 크기가 충분했나요? (검정력 분석 실행)
- 테스트가 충분히 오래 실행되었나요? (주간 주기, 최소 1~2 비즈니스 사이클 포착)
- 무작위화가 올바르게 이루어졌나요? (샘플 비율 불일치 확인)
- 테스트 기간 동안 외부 요인이 있었나요?

문제가 발견되면 표시 — 잘못 설계된 테스트의 결과는 오해를 불러일으킬 수 있음.

### Step 3: 결과 분석

**ab-test-analysis** 스킬을 적용합니다:

- **통계적 유의성**: p값 및 신뢰 구간 계산
- **효과 크기**: 변형 간 절대적 및 상대적 차이
- **실용적 유의성**: 효과가 비즈니스에 충분히 의미 있는가?
- **신뢰 구간**: 진정한 효과의 그럴듯한 범위는?
- **세그먼트 분석**: 데이터가 허용하는 경우, 사용자 세그먼트별 차별적 효과 확인

### Step 4: 분석 보고서 작성

## 참고 사항

- 통계적 유의성 ≠ 실용적 유의성 — 0.1% 향상은 충분한 데이터로 유의할 수 있지만 출시할 가치는 없을 수 있음
- 결과를 신뢰하기 전에 항상 샘플 비율 불일치 확인
- 신기 효과가 단기 결과를 부풀릴 수 있으므로 출시 후 2~4주 모니터링 권고
- 테스트의 검정력이 부족한 경우 올바른 답변은 보통 "효과 없음"이 아닌 "연장"
- 매출 지표의 경우 신뢰 구간을 사용하여 최선의 경우와 최악의 경우 비즈니스 영향 추정
- 데이터가 CSV로 제공된 경우 scipy.stats를 사용하는 Python으로 전체 분석 생성
