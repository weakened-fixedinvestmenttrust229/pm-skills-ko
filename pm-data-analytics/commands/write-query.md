# /write-query -- SQL 쿼리 생성기

필요한 데이터를 일반 언어로 설명하면 최적화된 SQL 쿼리를 생성합니다. 다양한 방언을 지원하며 업로드한 파일에서 스키마를 읽을 수 있습니다.

## 실행 예시

```
/write-query Show me daily active users for the last 30 days, broken down by plan tier
/write-query Find users who signed up last month but never completed onboarding
/write-query [스키마 다이어그램 업로드] What's the conversion rate from trial to paid by cohort?
```

## 워크플로

### Step 1: 질문 이해

사용자의 자연어 요청을 파싱하여 식별:
- 요청하는 데이터 (지표, 차원, 필터)
- 시간 범위 및 단위
- 그룹화 및 정렬 선호도
- 출력 기대값 (원시 데이터, 집계, 순위)

### Step 2: 스키마 확인

스키마가 있는 경우 (업로드된 다이어그램, DDL, 또는 설명):
- 요청을 특정 테이블 및 컬럼에 매핑
- 필요한 조인 식별

스키마가 없는 경우:
- 데이터베이스 유형 확인 (BigQuery, PostgreSQL, MySQL 등)
- 질문으로부터 합리적인 스키마를 추론하고 사용자에게 확인 요청
- 기본값으로 일반적인 SaaS 데이터 모델 관례 사용

### Step 3: 쿼리 생성

**sql-queries** 스킬을 적용합니다:

- 올바른 방언으로 SQL 쿼리 작성
- 가독성 및 성능 최적화
- 핵심 로직을 설명하는 주석 추가
- 복잡한 쿼리의 가독성 향상을 위해 CTE 추가
- 엣지 케이스 처리 (NULL, 시간대 고려, 중복 처리)

### Step 4: 제시 및 반복

```
## SQL 쿼리: [목적]

**방언**: [BigQuery / PostgreSQL / MySQL 등]
**사용된 테이블**: [목록]

### 쿼리
[주석이 포함된 SQL 코드 블록]

### 반환 데이터
[출력 설명: 컬럼, 행, 예상 결과 형태]

### 가정
- [스키마 가정 사항]
- [비즈니스 로직 가정 사항]

### 참고 사항
- [대용량 데이터셋에 대한 성능 고려 사항]
- [처리되었거나 표시된 엣지 케이스]
```

- "관련 지표에 대한 **동반 쿼리를 생성**해 드릴까요?"
- "이 쿼리를 중심으로 **대시보드를 구축**해 드릴까요?"
- "**코호트 분석** 버전이 필요하신가요?"

## 참고 사항

- 항상 SQL에 주석 포함 — PM이 쿼리를 이해해야 하는 분석가와 공유하기 때문
- 영리함보다 가독성 우선 — 중첩 서브쿼리보다 CTE 선호
- 대용량 데이터셋에서 느릴 수 있는 쿼리 표시 및 최적화 제안
- 요청이 모호한 경우 (예: "활성 사용자") 사용자에게 지표를 정확하게 정의하도록 요청
- 사용자가 어떤 데이터베이스를 사용하는지 모르는 경우 여러 방언으로 쿼리를 생성하도록 제안
